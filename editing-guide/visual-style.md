# Visual Style Notes

用途：说明个人网站的视觉方向和配色维护原则，避免后续维护时又改回课题组网站的稳重红灰风格。

## 当前视觉方向

这个个人站应比课题组网站更鲜活、更像个人作品集和学习笔记空间。当前采用偏“多巴胺”的多色系统：

- 珊瑚红：主要行动按钮、重点状态、部分卡片强调边。
- 柠檬黄：首页指标卡、辅助按钮、轻快提示。
- 亮蓝：研究和时间信息。
- 薄荷绿：工具、技能、实验类内容。
- 青蓝：项目和探索类内容。
- 紫罗兰：写作、博客、个人表达。

这些颜色集中定义在 `css/style.css` 的 `:root` 中：

```css
--color-brand: #ff3d71;
--color-sun: #ffd23f;
--color-mint: #40d98a;
--color-sky: #00b8d9;
--color-violet: #8b5cf6;
```

## 和课题组网站的区别

课题组网站更适合稳定、可信、低饱和的学术机构气质。个人站可以更年轻、明亮、具有探索感。

后续维护时不要直接复制课题组网站的：

- 深红 + 灰蓝主色系统。
- 大面积冷灰背景。
- 过于机构化的卡片风格。

## 修改配色时的边界

可以改：

- `:root` 里的颜色变量。
- 卡片的 `border-top` / `border-left` 强调色。
- 标签、按钮、导航高亮的背景色。
- `assets/images/site/hero-workspace.png` 首页背景图。

谨慎改：

- 正文字体颜色和段落颜色，必须保证可读。
- 首页 hero 的遮罩，必须保证白色标题能读清。
- 移动端导航背景，必须保持足够对比。

不要做：

- 全站只剩一种颜色的深浅变化。
- 大面积紫蓝渐变覆盖所有页面。
- 只靠颜色区分重要状态，而没有文字或结构提示。
- 为了鲜艳牺牲正文可读性。

## 首页背景图

首页背景图位置：

```text
assets/images/site/hero-workspace.png
```

它是本地生成的几何工作台风格图片，用来表达 research / code / reflection。替换时建议保持：

- 横向大图，推荐 16:9。
- 画面左侧不要过于复杂，因为首页文字主要在左侧。
- 色彩可以鲜活，但不要让标题和按钮看不清。
