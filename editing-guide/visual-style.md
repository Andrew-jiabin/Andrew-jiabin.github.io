# Visual Style Notes

用途：说明个人网站的视觉方向和配色维护原则，避免后续维护时又改回课题组网站的稳重红灰风格。

## 当前视觉方向

这个个人站应比课题组网站更鲜活、更像个人作品集和学习笔记空间。当前采用“舒服但有活力”的柔和多色系统：色相丰富，但饱和度不过分刺眼。

- 柔珊瑚红：主要行动按钮、重点状态、部分条目强调边。
- 温柠檬黄：首页快速入口、辅助按钮、轻快提示。
- 清亮蓝：研究和时间信息。
- 薄荷绿：工具、技能、实验类内容。
- 天蓝：项目和探索类内容。
- 柔紫罗兰：写作、博客、个人表达。

这些颜色集中定义在 `css/style.css` 的 `:root` 中：

```css
--color-brand: #ef6a7a;
--color-sun: #f4d35e;
--color-mint: #7fdca9;
--color-sky: #6ec6ff;
--color-violet: #a78bfa;
```

## 和课题组网站的区别

课题组网站更适合稳定、可信、低饱和的学术机构气质。个人站可以更年轻、明亮、具有探索感。

后续维护时不要直接复制课题组网站的：

- 深红 + 灰蓝主色系统。
- 大面积冷灰背景。
- 过于机构化的卡片风格。

## 修改配色时的边界

可以改：

- `:root` 里的颜色变量。
- 条目或少量卡片的 `border-top` / `border-left` 强调色。
- 标签、按钮、导航高亮的背景色。
- 首页 hero 彩虹渐变背景，定义在 `css/style.css` 的 `.home-hero`。

谨慎改：

- 正文字体颜色和段落颜色，必须保证可读。
- 首页 hero 的遮罩，必须保证白色标题能读清。
- 移动端导航背景，必须保持足够对比。

不要做：

- 全站只剩一种颜色的深浅变化。
- 大面积紫蓝渐变覆盖所有页面。
- 只靠颜色区分重要状态，而没有文字或结构提示。
- 为了鲜艳牺牲正文可读性。
- 为少量关键词单独做大卡片。关键词更适合放在正文末尾的轻量文字行里。

## 首页背景

首页不再使用具体内容图片，而是使用 CSS 写出的柔和彩虹渐变。相关位置：

```css
.home-hero
.home-hero::before
.home-hero::after
.hero-scrim
```

维护建议：

- 背景应保持抽象，不要放包含文字、图标、人物或具体科研内容的图片。
- 可以调整渐变颜色，但应保持柔和、通透。
- 必须保留足够深的 `.hero-scrim`，保证首页白色标题和简介可读。
- 手机端有单独的 `.home-hero` 和 `.hero-scrim` 覆盖规则，改桌面时也要检查手机端。
